To match the performance position with the state of the MIDI sound source

MIDI sound generator internally holds various values.

For example, play a part that is Domino, stop playing when you send Pitch Bend 4000, If you start playing from another place again, the pitch bend value inside the sound source will remain at 4000, It will be played at unintended intervals.

Of course not only the pitch bend but also the volume and the pan pot, There are many cases that you can not play as intended.

In Domino it is possible to match the current performance position and the state of the MIDI sound source to some extent.

To match the state, select [Match MIDI equipment state] from the [Play] menu.


Parameters whose states match are different depending on the sound source definition file, Program change events, volumes, pan pots, expressions, etc. are valid.

In addition, it is possible to adjust the state automatically.
To automate, select "Match MIDI equipment state automatically" from the "Play" menu.

Then, when you move the playing position, the [Match state of MIDI device] processing is executed automatically.

A Melo (Vol=50) Å® B Mero (Vol=60) Å® Rust Å® A Mero (Vol=50) Å® B Mero (Vol=60) Å® Sabi I think whether to insert an event on each occasion like that.

However, in this case, if you want to change the volume of A melody to 55, you have to fix two places.
(It's fine if you have two places but it will be tough when this becomes so much)

Therefore, using variables, "a_vol=50" and "b_vol=60" are defined in advance, If you specify it as the value of the Volume (Part Level) event, When you want to change the volume of A melody later, you can just modify "a_vol=55".

To actually define a variable, select [Variable Table] from the [Edit] menu.


Since all the variables existing in memory are displayed, Write the variable you want to use in the format "variable name=value" line by line.

(We will distinguish between full-width and half-width variable names, but we do not distinguish between upper case and lower case letters, or it will be lowercase.)

After that, you can use it by directly entering the variable name in the Vel/Value field of the control change event.


Since the effective range of this variable is all tracks in the file, if you simply use a name like a_vol there, If you want to do something similar on other tracks, you may be in trouble ^ ^ ;;

Incidentally, there are variable events, but this is an event that you can define and recalculate variables during playing. However, since it is not compatible with the [Match state of MIDI device] function, it is better to avoid using variable events when using that function.

To record in real time

You can import data played in real time from a MIDI keyboard etc. as data.
(It is not like recording audio from a microphone.)

To do real-time recording, please press the [Realtime recording] button on the tool bar.


The [Realtime recording] screen will be displayed, please set the playing range and tempo and press the [Record] button.


As the performance begins, please start inputting from a MIDI keyboard etc.

When the performance is over or you press the [STOP] button, the input contents are reflected on the screen.

To step recording

Step recording is a function to capture contents entered from the MIDI keyboard as data.
Unlike real-time recording, the note length (interval) is specified in advance, Recording is possible at free timing.

To step recording, please press the [Step Record] button on the tool bar.


The [STEP RECORD] screen will be displayed, so specify Step and start inputting from the MIDI keyboard.


If you press the [Next/Tie] [Previous] button during the sounding, you can increase/decrease the Gate of the input note.
(The same process is done by tilting the pitch bend instead of the [Next/Thail] [Previous] button.)

By pressing the [PC keyboard] button, you can step recording from the PC keyboard without using a MIDI keyboard.


You can change the shortcut key etc. from [Setting] button.

If you want to play a tone, press and hold down the [Ctrl] key and input.
(Events are not inserted while pressing [Ctrl] key.)

How do I adjust the end position of a song?

There is an event (End of Track) indicating the end position of the track for each track, The End of Track, which is the last in time on all tracks, is the end position of the song.

Even though the performance has ended, the performance has not ended for a while, On the contrary, when the end position of the song is too early (after writing it as a midi file) and the performance ends before the sound is finished, You can solve it by adjusting the playing position.

To adjust the end position of the song, set the position you want to be in the end position to the performance position.

Then select [Adjust End of Track] from the [Event] menu.


This will rearrange the End of Track events for all tracks to the playing position.
(Of course, if you have an event behind you can not relocate ^ ^;)

How do I use the backup function?

Editing While Domino is accessing a null pointer and falling, Families use dryers and microwave ovens, breakers fall, The file being edited may be lost due to such external factors.

Of course, it is okay if you keep [save] frequently, but sometimes the head does not turn to that extent.

Therefore, when you use the backup function, the saving process is automatically executed periodically.

To use the backup function, press the [Preferences] button on the toolbar,

Select [Backup] on the left side of the [Preferences] screen.

Check [Perform regular backup], specify the interval, file name, save destination and click the [OK] button.


This will save the file at the specified time.
