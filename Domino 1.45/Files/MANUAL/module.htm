Specification of sound source definition file

Overview

The sound source definition file is a file that defines the sound name of the sound source and various setting methods.
Domino manipulates the sound source based on this definition file.

The sound source definition file is stored in the "Module" folder located at the same position as Domino.exe.
If you create your own sound source definition file, or if you receive it from others, please create/copy it in this folder.

For each sound source, one file is created for the sound source definition file.
(It is not usual to define multiple sound sources with one file.)

main structure

The sound source definition file is described in XML (Extensible Markup Language) format.
(We use tag like HTML.)

The main structure of the definition file is as follows.
<? xml version="1.0" encoding="Shift_JIS"?>

< ModuleData (Writing basic information on sound source)>
(Describe options)
< InstrumentList>
(Writing tone color information)
</InstrumentList>
< DrumSetList>
(Describe drum set information)
</DrumSetList>
< ControlChangeMacroList>
(Write control change macro information)
</ControlChangeMacroList>
< TemplateList>
(Write template information)
</TemplateList>
< DefaultData>
(Write default data information)
</DefaultData>
</ModuleData>
(You can put tabs, blank lines, etc. freely.)

The rest is the definition file if you write the part written in parentheses.
I will explain each part.

Basic information of sound source

Describe the basic information of the sound source and definition file as an attribute of the ModuleData tag.

Example)
FileCreator="Takabo Soft" FileVersion="1.00" WebSite="http://takabosoft.com/"< ModuleData Name="SC-8850" Folder="Roland" Priority="100"

The attributes of the ModuleData tag are as follows.
(Attribute names and tag names are not case sensitive, and the order of attributes is free.)

attributewhen omittedmeaning
NameRequiredSpecify the sound source name.
FolderOn the screen when choosing a sound source on Domino, it enters the folder specified here.
Priority100Specify the priority (order) within the same folder. The smaller the value, the earliest it is.
FileCreatorSpecify the creator name of the definition file.
FileVersionSpecify the version of the definition file. It is a good idea to raise the version number if you modify the definition file.
WebSiteSpecify the distribution site or the homepage of the definition file.

The creator name and version information of the definition file can be checked from the [sound source definition file] information on the [Help] menu.

Options

Several options can be specified as elements of the ModuleData tag.

Example)
< RhythmTrackDefault Gate="1"/>
< ExclusiveEventDefault Data="f 0 h 41 h 10 h 42 h f 7 h"/>
< ProgramChangeEventPropertyDlg AutoPreviewDelay="0"/>
< ControlChangeEventDefault ID="0"/>

RhythmTrackDefault Tag
With the Gate attribute, specify the gate value of the note in the rhythm track.
ExclusiveEventDefault tag
Data attribute specifies the default data of exclusive event.
ProgramChangeEventPropertyDlg tag
AutoPreviewDelay attribute specifies the delay time (in milliseconds, default is 0) until automatic preview on the Program Change Event Properties screen.
Depending on the MIDI device, there is something that takes time to actually change the tone from sending a program change. In that case, if you specify this delay time, the correct tone will be sounded during automatic preview.
ControlChangeEventDefault Tag
ID attribute, specify the default ID when CC event is inserted.
According to the rule of XML, there is no need to close the tag (it completes with only one tag) Be sure to include a space and a slash at the end of the tag.

Vaule tag, Gate tag

Value tag, attributes of Gate tag are the same.
The Value tag defines the range of the Value value of the event.
The Gate tag defines the range of the event 's Gate value.
If not defined, the Value field/Gate field will be invalid.

The Value tag and the Gate tag can have multiple Entry tags (it does not matter) as table data.

Attributes of Value tag and Gate tag are as follows.
attributewhen omittedmeaning
Default0Specify the default value.
Min0Specify the minimum value.
Max127Specify the maximum value.
Offset0Specify a value to be added when actually sending data.
For example, panpot is specified in the range from -64 to +63, but when sending it to midi device, you have to send this value +64 before sending it. In such a case, specify Offset to 64.
NameSpecify the name of the value (not much on the screen).
TypeIf "Key" is specified, the button to refer to the keyboard appears on the property screen.
TableIDTo use the common table, specify the table ID here.

Entry tag

The Entry tag defines one table of data.
Entry tags are defined as elements of Gate tag, Value tag, Table tag.

Attributes of the Entry tag are as follows.
attributewhen omittedmeaning
LabelRequiredSpecify the text to be displayed.
ValueRequiredSpecify a value.

By defining the table in this way, the user will be able to select from the table as well as the value.

Memo tag

The Memo tag defines the memo of the control change macro.
It is used in the property change screen of the control change event etc.
If you want to put a line break in the sentence please write "\ n".

Data tag

The Data tag defines what information is sent to the MIDI device.

The content of the Data tag is defined by combining an instruction command with an at-sign and a value (fixed value/variable value).

Example)
< Data> @ NRPN 1AH#GL#VL#NONE</Data>

Separate instructions and values ??with spaces.
You can define as many as "@ command command value value ... @ command command value value ..."
(However, since it is the same period that is actually processed, it is not good to send many heavy processing at the same time.)

The list of command commands is as follows.
Send system exclusive. When using [and], it automatically inserts the checksum of the enclosing part into the] part before sending it.
SyntaxMeaning
@ PB [HighValue] [LowValue]Send pitch bend.
@ CP [Value]Send channel pressure.
@ PKP [Key] [Value]Send the polyphonic key pressure.
@ CC [ControlChangeNumber] [Value]Send control change.
@ SYSEX F 0 H ..... F 7 H
@ RPN [RPN MSB] [RPN LSB] [Data MSB] [Data LSB]Send RPN.
@ NRPN [NRPN MSB] [NRPN LSB] [Data MSB] [Data LSB]Send NRPN.

It is necessary to specify a fixed number of values ??following the command command.
To specify a fixed decimal value, enter "1" or "10".
To specify a fixed hexadecimal value, enter "1h", "10H", "0x20", etc.

Not only fixed values ??but also variable values ??can be used.

The list of variable values ??(and special values) is as follows.
Make values ??from 00H to 0FH based on theMake values ??up to 2FH based on the
SyntaxMeaning
#NONEIt does not have the value itself. It can be used in the [Data LSB] part of @ RPN.
#VLReference the lower 7 bits of the Value.
#VHRefers to the upper 7 bits of the Value.
#GLRefers to the lower 7 bits of the Gate value.
#GHRefers to the upper 7 bits of the Gate value.
#CHchannel.
#1 CHCreate values ??from 10 H to 1 FH based on the channel.
#2CHchannel.
#3 CHCreate a value from 30H to 3FH based on the channel.
#PCHIn the case of the A port, make values ??from 00H to 1FH based on the channel.
Similarly, B port makes 10H ~ 1FH, C port 20H ~ 2FH, D port 30H ~ 3FH value. After the E port, it will always be 00H. Please do not use in generic sound source definition file (because it is affected by port).
#1 RCHIn Roland's sound source manual, make the value indicated as "1x" (x is block number ? channel).
#2 RCHMake the value written as "2x" (x is block number ? channel) in Roland's sound source manual.
#4 RCHMake the value written as "4 x" (x is block number ? channel) in Roland's sound source manual.
#VF1Make values ??from 00H to 0FH based on 1 to 4 bits of Value.
#VF 2Make values ??from 00H to 0FH based on the value of 5 to 8 bits of Value.
#VF3Create values ??from 00H to 0FH based on the value of 9 to 12 bits of Value.
#VF4Make values ??from 00H to 0FH based on the value of 13 to 16 bits of the Value.

#RSCTRT 2

Table tag

Normal tables are defined using Entry tags as elements of Value tags and Gate tags, If you define a table as a common table, you can use the same table with multiple macros.

The Table tag can be defined just under the ControlChangeMacroList tag or just under the Folder tag.
A Table tag can have multiple Entry tags as elements.

The attributes of the Table tag are as follows.
attributewhen omittedmeaning
IDRequiredSpecify the table ID. Specify 0 or more and do not overlap.

Track tag

(Actually, the tracks are created in the order in which they are placed.)

You can have Mark tag, TimeSignature tag, KeySignature tag, CC tag, PC tag, Comment tag, Template tag, EOT tag as elements.
This tells you what events to include in the track.

The attributes of the Track tag are as follows.
attributewhen omittedmeaning

Ch1Specify the channel number. Please specify it in the range of 1 ~ 16.
ModeIf "Conductor" is specified, it becomes a conductor track (only one can be specified).
When "Rhythm" is specified, it will be treated as a rhythm track.
